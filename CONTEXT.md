# Контекст проекта «Соня» — текущее состояние

Telegram AI-бот цветочного магазина «ЦветоМира» (Набережные Челны). Подбирает
букеты из каталога, оформляет заявку и создаёт сделку в amoCRM. Подробности
архитектуры — в `README.md`.

## Что уже работает (в проде)

- **Бот**: @ChelnyCFlowersbot, развёрнут на VPS (Германия, AlmaLinux), Docker,
  `restart: unless-stopped`, путь на сервере `/opt/sonya`.
- **Консультация**: LLM через OpenRouter (модель `openai/gpt-4.1`), подбор букетов
  из WooCommerce Store API сайта `cvety-naberezhnye.ru`.
- **Диалог**: уточняет запрос → подбирает 2–3 варианта → собирает имя/телефон/способ
  получения → показывает сводку → после «да» создаёт заявку.
- **amoCRM**: заявки падают в воронку **«Цветы бот»** (pipeline 10919438, этап
  Initial Contact 85876394). Заполняются поля: Букет название, Букет цена,
  доставка/самовывоз; создаётся контакт с телефоном. Авторизация — long-lived token.
- После «Передаю флористу» бот замолкает (`state=handoff`).

## Что в процессе / открытые вопросы

- **Двусторонний чат менеджер↔клиент** ещё НЕ включён. Решается, каким способом:
  - **Нативный чат amoCRM (amojo)** — красивый чат-виджет в карточке сделки. Требует
    `channel_id` + `channel_secret`, которые amoCRM выдаёт только вручную (не
    self-service; подтверждено доками, поддержкой и проверкой API). Код моста готов:
    `app/crm/chat.py` (отправка с HMAC-подписью), `app/crm/webhook.py` (приём ответов).
    Нужен ещё HTTPS-домен для вебхука.
  - **Через комментарии сделки (поллинг)** — менеджер пишет примечание в сделке →
    клиенту в бота; ответ клиента → примечанием в сделку. Без канала/домена/HTTPS.
    Реализация начата (курсор примечаний в `app/db/storage.py`), НЕ завершена:
    осталось `services/note_poll.py`, правки `services/handoff.py` и запуск в `main.py`.

## Известные нюансы эксплуатации

- После правки `.env` на сервере использовать `docker compose up -d --force-recreate`,
  НЕ `restart` (restart не перечитывает env_file).
- `docker compose exec` не подхватывает env_file: для разовых тестов либо `-e VAR=...`,
  либо `cp .env data/.envtest` и в контейнере `set -a; . /app/data/.envtest`.
- Модель OpenRouter: старый slug `anthropic/claude-3.5-sonnet` мёртв (404); рабочие —
  `openai/gpt-4.1`, `anthropic/claude-sonnet-4.5`, `anthropic/claude-haiku-4.5`.
- Все секреты — в `.env` (НЕ в репозитории, см. `.gitignore`). Шаблон — `.env.example`.

## Деплой

```bash
# на сервере

```
Локальная разработка и переменные — см. `README.md` и `.env.example`.
