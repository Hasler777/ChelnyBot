# Соня — AI-консультант по цветам (Telegram + amoCRM)

Telegram-бот для магазина «ЦветоМира» (Набережные Челны). Подбирает букеты из
реального каталога `cvety-naberezhnye.ru`, оформляет заявку и **передаёт диалог
флористу**: создаёт сделку в amoCRM и переключается в режим живого чата — бот
замолкает, менеджер общается с клиентом из встроенного чата amoCRM.

## Как работает

```
Клиент ──Telegram──> Бот (aiogram)
                        │
        ┌───────────────┼────────────────────┐
        │ режим consult │ режим handoff       │
        ▼               ▼                     ▼
   LLM-консультант  WooCommerce        amoCRM:
   (OpenRouter)     Store API          • сделка (REST API)
   tool-calling     каталог/цены       • чат (amoJo, двусторонний)
```

- **LLM** — через OpenRouter (любая модель, OpenAI-совместимый API).
- **Каталог** — публичный WooCommerce Store API (`/wp-json/wc/store/products`),
  фильтрация по бюджету/запросу на стороне бота, кэш в памяти.
- **Заявка** — сделка amoCRM с полями (товар, цена, бюджет, способ получения).
- **Чат с менеджером** — amoCRM Chat API (amoJo): сообщения клиента уходят в amo,
  ответы менеджера прилетают вебхуком и пересылаются клиенту в Telegram.
- После передачи флористу бот по этому пользователю **молчит** (`state=handoff`).
  Возврат к боту — командой `/start`.

## Стек

Python 3.12, aiogram 3.x, aiohttp, openai (OpenRouter), aiosqlite.

## Запуск

1. `cp .env.example .env` и заполнить (см. ниже).
2. `pip install -r requirements.txt`
3. Настроить amoCRM: `python -m scripts.amo_bootstrap` — выведет ID воронок,
   статусов, кастомных полей и `amojo_id`. Впишите их в `.env`. Затем создайте
   кастомные поля сделки (товар/цена/бюджет/способ получения/источник) в amoCRM,
   снова запустите скрипт и впишите их ID в `AMO_CF_*`.
4. Подключить канал чата: создайте «Чат» в кабинете интеграций amoCRM, получите
   `AMOJO_CHANNEL_ID` и `AMOJO_CHANNEL_SECRET`, укажите URL вебхука
   (`https://ваш-домен{WEBHOOK_PATH}`), затем
   `python -m scripts.amo_bootstrap connect` → впишите `AMOJO_SCOPE_ID` и
   `AMOJO_ACCOUNT_ID`.
5. Запуск: `python -m app.main` (локально) или `docker compose up -d` (на VPS).

> Вебхук amoJo требует HTTPS — на VPS поставьте reverse-proxy (Caddy/nginx) с TLS
> перед портом `8080`. Локально для теста можно `ngrok http 8080`.

## Переменные окружения

Все ключи и их назначение — в `.env.example`.

## Конфигурация без amoCRM

Если `AMO_*`/`AMOJO_*` не заполнены, бот работает как консультант: ведёт диалог,
подбирает букеты и доходит до «Передаю флористу», но сделка/чат не создаются
(в логах — предупреждение). Удобно для первого теста только связки бот+LLM+каталог.

## Замечания по amoJo

Подпись запросов amoJo (`Content-MD5` + `X-Signature` = HMAC-SHA1) и формат тела
вебхука сверяйте с актуальной документацией amoCRM Chat API — детали (имена полей
`conversation_id`/`client_id`) могут отличаться между версиями. Парсер вебхука в
`app/crm/webhook.py` написан устойчиво (ищет ID в нескольких местах), но при
интеграции стоит залогировать первое реальное событие и при необходимости
подправить `_extract`.
